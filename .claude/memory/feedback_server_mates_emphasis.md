When writing in-game player-facing text (Patchouli book, FTB Quests, signs, etc.), emphasize **the community of players on the server** as the source of help and life on the server. The user explicitly does not want to position himself as a special host or authority figure — his framing is "I'm no one special, just another server mate."

**Why:** Multiplayer is the most important pillar of this pack to the user personally. He wants players to feel like they joined a *community*, not a server hosted by him. Positioning him as host/expert/author cuts against that feeling.

**How to apply:**
- Default any "ask me" or "ping the host" line to "ask anyone in voice chat" / "ask a server mate."
- The user can still sign personal notes (~Wesley) — that's warmth, not authority. The thing to avoid is implying he is *the* helper or expert.
- In multiplayer chapter / FAQ-type content, lead with the social fabric: voice chat, doing things together, asking each other for help. Don't make it a list of features.
- $(bold)Redact role-establishing phrasing entirely$() — even denials count. Don't write "I'm not the host so much as another player," "I built this for us," "I'm not trying to teach you" etc. Even negating the host role still names it. Just state the expected behavior/norm directly. Personal warmth ("you're someone I care about") and signature are fine; positioning the user as builder/host/expert/intent-haver is not.
